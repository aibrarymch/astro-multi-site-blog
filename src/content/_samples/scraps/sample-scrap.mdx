これはスクラップ機能のテストです。

Zennのスクラップのように、気軽にメモを追記していけます。

思いついたことを気軽にメモとして残しておける機能です。ブログ記事ほど構成を考えなくても、断片的な情報をそのまま書き留めておけるのが便利ですね。

あとから振り返ったときに、当時の思考過程がわかるのも良いポイントです。
</ScrapEntry>

<ScrapEntry datetime="2024-05-25T14:30:00">
コードブロックも使えます:

<CodeBlock lang="bash" copyMode="button">mkdir astro-blog
cd astro-blog
npm install
npm run dev</CodeBlock>

上記のコマンドで開発環境が立ち上がります。ローカルホストにアクセスすると、ホットリロードが効いた状態でプレビューできます。

- リスト項目1
- リスト項目2
- リスト項目3

リストの前後にも段落を置いてみました。余白の確認に使えます。
</ScrapEntry>

<ScrapEntry datetime="2024-05-25T18:00:00">
## 見出しのテスト

見出し直後の段落です。ここの上余白は1.5emになっているはずです。

通常の段落間の余白は2emです。この段落と上の段落の間隔が、見出し直後よりも広くなっていれば正しく動作しています。

### 小見出しも試す

小見出し直後の段落も同様に1.5emの余白になります。

こちらは通常の段落間なので2emの余白です。見出し直後との違いが確認できるはずです。
</ScrapEntry>

<ScrapEntry datetime="2024-05-26T09:15:00">
翌日の追記です。

新しい知見があればどんどん追加していきましょう。

段落を複数並べて余白を確認します。スクラップ内でも記事本文と同じように、段落間の余白が適用されていることを確認できます。

最終的にはデザインの統一感が大事ですね。記事でもスクラップでも、読みやすい余白が保たれていると嬉しいです。
